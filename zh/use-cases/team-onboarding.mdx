## 挑战

你有一个很棒的 Alter 设置：
- 适合您团队工作流程的自定义操作
- 精心配置的设置
- 包含重要文件的共享Workspaces
- 最适合您任务的特定模型

但每次有新人加入时，他们都必须从头开始解决一切问题。他们：
- 使用错误的型号
- 不知道您的自定义操作
- 重新创建您已经完成的工作
- 输出不一致

## 解决方案：分享您的设置

Alter 可以轻松共享配置，以便新团队成员在第一天就可以高效工作。

## 分步团队入职

<Steps>
  <Step title="创建基线配置">
    记录您团队的标准设置：
    - **推荐模型**（例如，“使用 Claude 进行代码审查，使用 Gemini 进行创意任务”）
    - **默认温度**（例如，0.7 用于平衡创造力）
    - **语音热键**（标准化 `Fn` 或自定义键）
    - **Workspaces的文件命名约定**
  </Step>
  
  <Step title="构建入门Workspaces">
    创建具有共同上下文的共享Workspaces：
    - **代码库Workspaces：** 包括您的主项目文件夹
    - **文档Workspaces：** 包括规格、要求、风格指南
    - **知识库Workspaces：** 包括过去的决策、常见问题解答、维基百科
    
    <Tip>
      Workspaces 存储在 `~/Library/Application Support/Alter/Workspaces/` 中。您可以直接共享 `.alterworkspace` 文件或记录如何创建它们。
    </Tip>
  </Step>
  
  <Step title="共享自定义操作">
    导出团队最有用的操作：
    1. 打开**动作编辑器**（从菜单栏）
    2. 选择一个操作
    3. 单击**导出**（或按`Cmd + E`）
    4.保存`.json`文件
    5. 与您的团队分享
    
    新成员可以在操作编辑器中使用 `Cmd + O` 导入这些内容。
  </Step>
  
  <Step title="创建团队资料模板">
    共享个人资料描述模板，新成员可以将其粘贴到“设置”>“个人资料”中：
    
    ```
    Role: [Job title]
    Team: [Team name]
    Communication style: [Direct/casual/formal]
    
    Common tools we use:
    - GitHub repo: [URL]
    - Project management: [Linear/Asana/Jira]
    - Documentation: [Notion/Confluence]
    
    Technical context:
    - Primary language: [Python/JavaScript/etc]
    - Framework: [React/Django/etc]
    - Key dependencies: [List main libraries]
    ```
  </Step>
</Steps>

## 示例：开发团队如何使用它

**设置：**
- **标准型号：** Claude Sonnet 适用于大多数任务
- **代码审查操作：** 预先配置以检查安全问题、测试和文档
- **共享Workspaces：** 主要代码库 + API 文档 + 部署指南
- **团队简介：** 每个人都包含 GitHub 存储库结构和关键依赖项

**结果：**
当新开发人员加入时：
1.他们安装Alter
2.导入团队的Actions（5分钟）
3. 克隆共享Workspaces（2 分钟）  
4.粘贴个人资料模板（1分钟）
5. 他们立即变得高效——他们可以问“身份验证系统是如何工作的？”并获得上下文答案
