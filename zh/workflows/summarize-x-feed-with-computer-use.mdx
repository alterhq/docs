当 API 访问受到限制并且您仍然需要从 X (Twitter) feed 中快速了解情况时，请使用此工作流程。

## 这可以帮助您做什么

您可以将您的提要视为 Alter 从屏幕上读取的实时源。 Alter 不是从 API 中提取数据，而是总结浏览器中当前可见的内容，并帮助您迭代地构建摘要。

## 适用于

- 应用程序：浏览器中的 X
- 工具类型：计算机使用（基于屏幕）
- 权限：辅助功能和屏幕录制

## Alter 今天可以做什么

Alter 可以读取可见的提要内容、总结屏幕上的帖子、提取重复出现的主题以及将屏幕上的多个批次合并为一个摘要。

## Alter 还没有做什么

- 此工作流程不使用官方 X API。
- 它无法总结当前不可见的帖子，除非您滚动并再次询问。
- 它不是完整的 X 帐户自动化流程（发布、审核或分析管道）。

## 真实的工作流程

<Steps>
  <Step title="Capture batch 1">
    打开您的时间线并要求 Alter 总结可见的帖子。
  </Step>
  <Step title="Capture batch 2">
    向下滚动并要求针对新帖子的第二个摘要。
  </Step>
  <Step title="Merge into digest">
    要求 Alter 将两个摘要合并为一份简明报告。
  </Step>
</Steps>

## 复制/粘贴提示

- “总结我的 X feed 中当前可见的推文。”
- “列出可见帖子中排名前 5 的主题，每个主题都有一行证据。”
- “向下滚动，捕获下一批，并仅总结新主题。”
- “将最后两份提要摘要合并为一份简明的每日摘要。”

## 如何启用

<Steps>
  <Step title="Enable Computer Use tools">
    打开Alter -> 单击三个点 -> **工具管理器** -> **本地工具** -> 启用**Alter** 工具（计算机使用）。
  </Step>
  <Step title="Grant required macOS permissions">
    在 Alter 设置中，允许 **辅助功能** 和 **屏幕录制**，以便 Alter 可以读取应用程序窗口并与之交互。
  </Step>
  <Step title="Run a quick test on X">
    打开您的 X 时间线并询问：“总结屏幕上可见的帖子。”
  </Step>
</Steps>

## 期待什么

- 最佳质量来自短迭代循环：总结 -> 滚动 -> 总结 -> 合并。
- 如果定位了错误的窗口，请将浏览器窗口置于前面或要求 Alter 列出窗口并定位正确的窗口。
