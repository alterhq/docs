<Info>
  **改变操作**是可自定义的人工智能工作流程，可以让您为特定任务创建专门的助手。结合AI指令、语音触发、工具和条件执行，以自动化Mac上的任何操作。
</Info>

## 什么是更改操作？

**更改操作**是一种高级自动化，它结合了：

- **AI指令** - 系统提示定义AI的行为方式
- **执行** - 语音命令、热键或条件执行
- **上下文** - 访问选定的文本、文件、活动应用程序
- **工具** - 配置与您的 Mac 和 2000 多个外部服务的集成
- **输出** - 结果如何展示（降价、代码、内联插入）

将Alter Action视为针对特定任务定制的“专业人工智能助手”。

## 打开动作编辑器

### 方法1：菜单栏（最快）
1. 单击菜单栏中的 **更改菜单图标**（右上角）
2. 选择**动作编辑器**

![菜单栏](/images/tools/menu-bar-action-editor.webp)

### 方法 2：选项菜单
1. 打开更改窗口
2.点击右上角的********（三个点）
3. 选择**动作编辑器**

![选项菜单](/images/tools/options-menu-action-editor.webp)

###方法3：键盘快捷键
按 **⌘ E** (Command + E) 直接打开动作编辑器

<Tip>
  一旦您记住了键盘快捷键，它就是最快的方法。
</Tip>

## 动作编辑器演练

动作编辑器有 7 个主要部分，每个部分控制动作的不同方面。使用左侧面板在选项卡之间导航。

### 1. 常规设置

配置基本元数据和执行上下文。

![常规选项卡](/images/tools/action-editor-01-general.webp)

** 字段：**
- **名称** - 可执行的操作名称（例如“人类代码解释器”）
- **描述** - 此操作的作用（任选）
- **类别** - 选择类型（代码、生产力、业务等）组织操作
- **工作空间** - 分配特定的工作空间（任选）
- **包含个人数据** - 在AI上下文中包含用户个人数据信息

**可见性选项：**
- **仅在运行时显示** - 仅当应用程序正在运行时才会显示操作
- **仅在活动时显示** - 仅当应用程序具有焦点时才显示操作
- **工作空间上下文** - 选择哪些工作空间可以看到此操作

<Note>
  每个操作都会自动获得一个唯一的UUID。
</Note>

### 2.系统提示

定义模型的AI指令和参数。

![系统提示选项卡](/images/tools/action-editor-02-systemprompt.webp)

**系统提示** - 描述AI行为和专业知识的核心AI指令。

**示例：**
```
# IDENTITY AND PURPOSE
You are an expert software engineer who explains code clearly
and accessibly to developers of all skill levels.

# STEPS
1. Analyze the provided code
2. Identify its purpose and key components
3. Explain each section in simple terms

# OUTPUT INSTRUCTIONS
- Use markdown formatting
- Include code examples where helpful
- Keep language clear and non-technical
```

**参数** - 定义用户可以在执行前自定义的输入字段（文本输入、下拉选择等）。

### 3. 用户提示

发送到 AI 的用户的提示模板。

![用户提示选项卡](/images/tools/action-editor-03-userprompt.webp)

**使用上下文变量：**

将变量包装在 `{{ }}` 中以插入动态内容：

```
Please explain this code:

{{ textSelection }}

Explanation detail: {{ detailLevel }};
```

**可用的上下文变量：**
- `{{ textSelection }}` - 当前选择的文本
- `{{ filePath }}` - 所选文件的路径
- `{{ clipboard }}` - 剪贴板内容
- 您在系统提示中定义的自定义参数

### 4. 何时展示

控制此操作出现的时间和位置。

![何时显示选项卡](/images/tools/action-editor-04-whentoshow.webp)

**能见度条件：**

- **已安装的应用程序** - 仅在特定应用程序（VSCode、Xcode、Safari 等）中显示操作
- **应用程序正在运行** - 操作要求应用程序正在运行
- **具有程序焦点的应用程序** - 只有当应用程序位于前台时操作才有效
- **Web浏览器选项卡包含** - 浏览特定域时显示操作
- **有内容** - 只有当上下文可用时才显示操作

### 5. 自动化

配置执行行为、计划和后台处理。

![自动化选项卡](/images/tools/action-editor-05-automation.webp)

**字段：**

- **后台作业** - 在后台运行操作而不显示UI
- **时间表** - 在特定时间执行操作（例如，“每周一上午 9 点”）
- **快速操作** - 完成后自动执行下一个“快速操作”

将此用于操作链接：第一个操作 → 输出 → 下一个操作自动执行。

### 6. 工具

可用于此操作的工具范围。

![工具选项卡](/images/tools/action-editor-06-tools.webp)

**工具配置：**

- **禁用工具** - 关闭单个工具以限制功能
- 空时：所有工具都可用
- 配置后：仅选定的工具可用

**为什么选择范围工具？**
- 减少人工智能决策开销（关注相关工具）
- 加快行动执行速度
- 提高安全性（限制可以访问的操作）

### 7. 模型

选择AI模型并调整创造力水平。

![模型选项卡](/images/tools/action-editor-07-model.webp)

**字段：**

- **模型** - 为此选择操作特定的AI模型
  - 留空以使用用户的默认模型设置
  - 或选择：Claude、GPT-4、本地模型等。

- **温度** - 调整创造力与精度
  - **0.0** - 确定性（一致、可预测）
  - **0.5** - 平衡（默认）
  - **1.0** - 创意（不可预测，探索性）

## 常见操作示例

### 示例 1：代码解释器

**设置：**
- **名称：** 解释代码
- **类别：**代码
- **安装的应用程序：** VSCode、Xcode
- **系统提示：**您是一位代码讲解的专家开发人员
- **用户提示：** `Please explain this code:\n\n{{ textSelection }}`
- **消费者：**降价
- **温度：** 0.5

### 示例 2：电子邮件草稿创建器

**设置：**
- **姓名：** 电子邮件草稿
- **类别：**生产力
- **何时显示：** 邮件应用程序处于活动状态
- **系统提示：** 您是专业的商务沟通者
- **用户提示：** `Transform this into a professional email:\n\n{{ textSelection }}`
- **工具：** 仅启用邮件
- **消费者：**就地
- **温度：** 0.6

### 示例 3：预定每日简报

**设置：**
- **名称：**每日简报
- **类别：**生产力
- **自动化：** 每天上午 9:00 的时间表
- **后台作业：**已启用
- **系统提示：** 生成每日优先事项简介

## 语音触发器

分配语音热键以免提触发操作：

<Steps>
  <Step title="系统设置">
    转到 **设置 > 语音触发器**
  </Step>
  <Step title="添加触发器">
    单击 **+ 添加触发器**
  </Step>
  <Step title="选择操作">
    从下拉列表中选择操作
  </Step>
  <Step title="指定短语">
    输入触发短语（例如，代码解释器的“解释”）
  </Step>
  <Step title="节省">
    单击“保存”即可
  </Step>
</Steps>

**用法：** 按住语音热键 + 说出触发短语 → 操作在当前上下文中执行

## x-callback-url 集成

使用特殊的 URL 从**任何应用程序**触发更改操作。这实现了强大的跨应用程序自动化工作流程。

### x-callback-url 是什么？

x-callback-url 是一种应用程序间通信协议，允许应用程序通过一个 URL 相互触发操作。

- **入站** - 从其他应用程序（快捷方式、注释、Safari等）触发更改操作
- **出站** - 在Alter中创建可打开其他应用程序（Bear、OmniFocus、Drafts等）的链接

### 获取操作的回调URL

<Steps>
  <Step title="打开动作编辑器">
    按 **⌘ E**
  </Step>
  <Step title="选择操作">
    选择您要公开的操作
  </Step>
  <Step title="转到常规">
    导航到“常规”选项卡
  </Step>
  <Step title="找到 URL Callback">
    找到“URL回调”部分
  </Step>
  <Step title="复制网址">
    复制`alter://` URL
  </Step>
</Steps>

**格式：**
```
alter://action/{action-id}?input=VALUE&param2=VALUE
```

### 从其他应用程序触发更改操作

**示例 1：从 Safari 进行网络搜索**
```
alter://action/ask-web?input=What+is+Alter+MacOS
```

**示例 2：带输入的代码解释器**
```
alter://action/explain-code?input=const+hello+%3D+%28%29+%3D%3E+%7B%7D
```

**示例 3：在 Apple 快捷方式中使用**
1.创建一个新的便捷方式
2.添加“打开URL”动作
3. 粘贴您的`alter://` URL
4.运行快捷方式来触发操作

## 故障排除

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="操作未出现在菜单中">
    - 检查**何时显示**设置
    - 验证应用程序正在运行/活动（如果已配置）
    - 确保操作已**启用**
  </Accordion>

<Accordion icon="triangle-exclamation" title="动作运行但输出较差">
    - 查看**系统提示** — 更具体
    - 调整**温度**（降低精度，提高创造力）
    - 检查**用户提示** — 确保变量正确
  </Accordion>

<Accordion icon="microphone" title="语音触发不起作用">
    - 验证热键是否在“设置”中配置
    - 检查麦克风权限
    - 首先使用手动触发器进行测试
  </Accordion>

<Accordion icon="lock" title="Tools 在操作中不可用">
    - 转到**工具**选项卡
    - 验证工具没有被禁用
    - 检查该工具的系统权限
  </Accordion>
</AccordionGroup>
