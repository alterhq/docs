## 与您的应用程序交互

### AppSense - 了解您的 Mac

与传统的屏幕获取方法不同，AppSense 直接从 macOS 应用程序读取实际数据，从而允许改变访问屏幕上可见外部的信息。

<CardGroup cols={2}>
  <Card title="深度整合" icon="plug">
    访问应用程序数据，而不仅仅是屏幕截图
  </Card>
  <Card title="情境意识" icon="brain">
    自动了解您正在做什么
  </Card>
</CardGroup>

#### 当前应用程序

改变自动建议使用活动窗口作为上下文：

<Steps>
  <Step title="寻找建议">
    点击提示框下方的**当前应用**
  </Step>
  <Step title="或者使用键盘">
    按`↓` 然后按`Enter` 将其选中
  </Step>
</Steps>

## 跟随模式

接下来的模式是Alter的一项核心功能，可以在聊天会话期间实时同步活动应用程序、文档和文件中的数据。

### 它是如何运作的

<AccordionGroup>
  <Accordion icon="refresh-cw" title="实时监控">
    接下来模式持续监控您正在使用的应用程序和文件的状态，在每次对话时为 Alter 提供有关当前工作流程的全面上下文。
  </Accordion>
  
<Accordion icon="lightbulb" title="实际例子">
    正在制作着陆页副本吗？当您进行更改并更改您的营销作业时，更改后会看到您所做的更改，从而消耗您解释您所做的事情。
  </Accordion>
</AccordionGroup>

### 使用跟随模式

<Steps>
  <Step title="添加上下文">
    将您希望包含到上下文列表中的应用程序或文档添加到上下文列表中
  </Step>
  <Step title="开始聊天">
    改变自动跟踪所选应用程序的状态
  </Step>
  <Step title="提出问题">
    即使当前在屏幕上不可见，也可以询问内容
  </Step>
  <Step title="继续工作">
    与选定的应用程序交互或在聊天时更新文档
  </Step>
</Steps>

<Tip>
  **专业提示：**以下模式非常适合长篇内容创建。将您的文档添加到上下文中一次，然后继续编写，同时更改提供实时反馈。
</Tip>

## 工作空间

工作区是一个基础的功能，可以让您保存、组织和自动刷新高效文件夹、文件和应用程序的集合，以实现的AI辅助工作流程。

<CardGroup cols={3}>
  <Card title="更快的响应" icon="zap">
    仅检索与您的查询相关的文件
  </Card>
  <Card title="成本效益" icon="credit-card">
    通过智能索引降低处理成本
  </Card>
  <Card title="处理大型项目" icon="folder-open">
    高效处理大型文件夹和代码库
  </Card>
</CardGroup>

### 工作区如何工作

工作区创建您所选资源的索引图。文件会自动转换：

|文件类型 |转换 |
|------------|------------|
| **PDF** | 带标签的 Markdown 页面标记 |
| **DOCX** | Markdown 格式 |
| **XLSX** | Markdown/CSV 格式 |
| **图片** | OCR 文本提取 |
| **目录** | 减压处理 |

<Note>
  工作区尚不支持音频和视频文件。首先通过进入Alter来重新记录它们。
</Note>

### 编辑工作区中的文件

<Tabs>
  <Tab title="单一编辑">
    使用自然语言编辑或创建单个文件：
    - 修改代码
    - 更新文档
    - 重构函数
    - 从头开始创建新文件
  </Tab>
  
<Tab title="多重编辑">
    同时编辑多个文件：
    - 非常适合重构
    - 更新跨文件的导入
    - 跨代码库的一致更改
  </Tab>
  
<Tab title="重击 Tool">
    执行 bash 命令进行高级操作：
    - 运行脚本
    - 批量操作
    - 复杂的工作流程自动化
  </Tab>
</Tabs>

<Info>
  **安全第一：** 所有编辑操作均使用安全带策略进行沙箱处理，并进行安全检查以确保安全执行。
</Info>

### 创建工作空间

<Steps>
  <Step title="打开集线器">
    按`Shift + Cmd + H`或从菜单栏中选择
  </Step>
  <Step title="导航至 Workspaces">
    在侧边栏中找到“工作区”部分
  </Step>
  <Step title="创造新的">
    单击工作区选择器中的 **++** 按钮
  </Step>
  <Step title="添加文件">
    使用“添加文件”按钮选择文件夹和文件
  </Step>
</Steps>

### 工作区存储

工作空间存储在本地：
```
~/Library/Application Support/Alter/Workspaces/
```

每个工作区包含：
- 工作区定义 (`.alterworkspace`)
- 您的文件和文件夹
- 缓存转换后的文件以便快速访问

## 更改操作 - 工作流程

更改操作是使用触发器和指令配置的高级提示。

### 创建动作

<CardGroup cols={2}>
  <Card title="动作编辑器" icon="edit">
    可从菜单栏访问以进行完全控制
  </Card>
  <Card title="快速创建" icon="plus">
    在提示框中输入“创建>更改操作”
  </Card>
</CardGroup>

<Note>
  内置操作无法编辑，但可以复制和自定义。
</Note>

### 定制型号选择

将特定的人工智能模型分配给各个动作以进行精确控制。

#### 为什么使用不同的模型？

<AccordionGroup>
  <Accordion icon="shield" title="隐私第一的工作流程">
    使用本地模型执行敏感任务，同时保留云模型执行一般查询
  </Accordion>
  
<Accordion icon="gauge" title="专业性能">
    分配快速、工具优化的模型（例如Qwen3）以实现自动化，同时使用创意模型进行写作
  </Accordion>
  
<Accordion icon="piggy-bank" title="成本优化">
    使用轻量级本地模型来完成简单任务，并为复杂操作保留强大的模型
  </Accordion>
</AccordionGroup>

#### 设置

<Steps>
  <Step title="打开动作编辑器">
    单击更改窗口中的三个点
  </Step>
  <Step title="选择动作">
    选择要自定义的更改操作
  </Step>
  <Step title="转到高级">
    找到“高级”部分
  </Step>
  <Step title="选择型号">
    选择此操作的具体型号
  </Step>
  <Step title="节省">
    该操作现在使用您选择的模型
  </Step>
</Steps>

### 操作级工具定制

准确选择每个操作可以使用哪些工具。

＃＃＃＃ 好处

<CardGroup cols={2}>
  <Card title="执行速度更快" icon="zap">
    将工具限制为仅需要的工具
  </Card>
  <Card title="更好的准确性" icon="target">
    防止人工智能选择不相关的工具
  </Card>
  <Card title="情境感知" icon="brain">
    针对每个操作创建专门的工作流程
  </Card>
  <Card title="成本效益" icon="credit-card">
    减少代币使用
  </Card>
</CardGroup>

＃＃＃＃ 配置

<Steps>
  <Step title="打开动作编辑器">
    单击三个点→“动作编辑器”
  </Step>
  <Step title="选择动作">
    选择更改操作
  </Step>
  <Step title="查找 Tools 部分">
    查看“高级”设置
  </Step>
  <Step title="选择Tools">
    选择特定工具或禁用所有工具
  </Step>
</Steps>

#### 工具集策略

- **最少：** 用于集中行动的一两个工具
- **分组：**特定工作流程的相关工具
- **完全访问：** 用于通用操作的所有工具
- **无工具：** 用于创意或分析任务

<Tip>
  **专业提示：** 首先禁用新操作的所有工具，然后仅添加您需要的工具。这可确保最佳性能并防止意外的工具使用。
</Tip>
