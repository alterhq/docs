title: "Tools 不工作"
description: "当 Alter 不使用您连接的Tools和应用程序时修复集成"
---
<Note>
  **快速检查：** 您选择的AI模型是否支持Tools？一些较小的模型不具有Tools调用功能。
</Note>

##发生了什么事？

您要求 Alter 执行“向 John 发送消息”或“添加日历事件”之类的操作，但是：
- 人工智能以文本响应而不是采取行动
- 目标应用程序中没有任何反应
- 您会收到“我无权访问该内容”或类似的回复

## 为什么会发生这种情况

<CardGroup cols={2}>
  <Card title="型号不支持Tools" icon="brain">
    一些人工智能模型无法调用Tools——它们只能生成文本
  </Card>
  <Card title="Tool 未启用" icon="toggle-off">
    该Tools在 Tools Manager 中可用但未打开
  </Card>
  <Card title="提示格式错误" icon="message">
    人工智能需要关于使用什么Tools以及如何使用的明确说明
  </Card>
  <Card title="全局 Tools 关闭" icon="toggle-off">
    所有Tools均通过 Active Tools 中的全局切换禁用
  </Card>
</CardGroup>

## 逐步修复

<Steps>
  <Step title="切换到支持Tools的模型">
    并非所有模型都可以使用Tools。尝试以下其中一项：
    - **Alter Best**（我们的智能路由器）
    - **Gemini 2.0 闪存**
    - **Claude十四行诗**
    - **GPT-5**
    
    在提示框中按`/`可以切换机型。
  </Step>
  
  <Step title="检查全局Tools切换">
    在 Tools Manager 中，有一个主开关可以一次性禁用**所有**Tools：
    1. 点击Alter中的**三个点** → **Tools Manager**
    2. 在 **Active Tools** 部分查找全局切换
    3. 确保其已**打开**（已启用）
    
    如果关闭此选项，即使启用了您的个人Tools，它们也将无法工作。
  </Step>

  <Step title="启用正确的Tools">
    全局切换打开后，启用特定Tools：
    1. 在Tools Manager中，找到您想要的Tools（消息、日历等）
    2. 将其**打开**
    3. 检查是否需要额外设置（如登录）
  </Step>
  
  <Step title="明确提出您的要求">
    而不是：*“告诉约翰我要迟到了”*
    
    尝试：*“使用消息Tools向 John Smith 发送一条消息，说‘我迟到了 10 分钟’”*
    
    你说得越具体，人工智能就越能理解。
  </Step>
  
  <Step title="对于苹果快捷方式">
    如果使用 Apple 快捷方式：
    1. 打开**快捷方式**应用程序
    2. 编辑快捷方式
    3.添加“接收输入”并将其设置为接受文本
    4.保存快捷方式
    5. 在 Alter 中重试
  </Step>
</Steps>

## 测试一下

尝试这个简单的测试：
1. 确保您使用的是支持Tools的模型
2.启用**系统通知**Tools
3. 说：“发送系统通知‘测试成功’”
4. **成功：** 您应该会看到出现 macOS 通知！

<Tip>
  **从简单开始：** 在尝试复杂的集成之前，使用系统通知等基本Tools进行测试。这有助于确定这是一般Tools问题还是特定于某个集成的问题。
</Tip>

<Tip>
  **尝试 Flow：** 如果您不确定要使用哪个Tools或在选择Tools时遇到问题，请启用 [Flow](/guides/use-flow-orchestration) - Alter 的Tools编排器，它会自动为您选择正确的Tools。
</Tip>

## 还是没有工作？

有些模型比其他模型更擅长使用Tools。如果一种模型不能很好地使用Tools：
1.切换到**Alter Best**（它会自动选择有能力的型号）
2. 或者尝试 **Gemini 2.0 Flash** 或 **Claude Sonnet**
