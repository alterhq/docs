title: "麦克风不工作"
description: "修复 Alter 不转录您的语音时的语音输入和听写问题"
---
<Note>
  **快速检查：** 您的 Mac 麦克风在语音备忘录等其他应用程序中是否正常工作？如果不是，这是系统问题，而不是 Alter 问题。
</Note>

##发生了什么事？

您按住麦克风按钮或按语音热键，但是：
- 当你说话时什么也没有发生
- 听写开始但没有文本出现
- 录音指示器显示但保持空白

## 常见原因

<CardGroup cols={2}>
  <Card title="缺少权限" icon="shield">
    Alter 无权访问您的麦克风
  </Card>
  <Card title="语音处理器错误" icon="microphone">
    未下载或配置本地语音模型
  </Card>
  <Card title="语言不匹配" icon="language">
    自动语言检测无法识别您的口音
  </Card>
  <Card title="输入设备错误" icon="speaker">
    Alter 使用了错误的麦克风或系统默认设置不起作用
  </Card>
</CardGroup>

## 如何修复它

<Steps>
  <Step title="检查麦克风权限">
    1. 在 Mac 上打开 **系统设置**
    2. 转到 **隐私与安全** → **麦克风**
    3. 确保 **Alter** 在列表中启用
    4.如果它已经打开，请尝试将其关闭然后重新打开
  </Step>

  <Step title="选择正确的输入设备">
    默认情况下， Alter 使用系统默认麦克风，但 macOS 并不总是正确处理此问题：
    1.在Alter界面中寻找**控制块**（或者查看菜单栏）
    2. 单击打开设备选择器
    3. 从列表中选择您的特定麦克风
    4. 再次尝试说话

    这解决了以下问题：
    - Alter 尽管已授予权限但未拾取任何音频
    - 会议录音仅捕获其他参与者而不是您
    - 低质量或间歇性音频输入
  </Step>

  <Step title="验证语音处理器设置">
    1. 在Alter中，打开**设置** (`Cmd + ,`)
    2. 转到 **听写** 选项卡
    3. 检查 **语音处理器** 设置为：
       - **云**（以获得最佳准确性）
       - **本地（Apple）**（出于隐私考虑）
       - 如果您想要离线处理，或者下载 Whisper 模型
  </Step>
  
  <Step title="明确设置语言">
    如果自动检测不起作用：
    1. 在**设置** → **听写**
    2. 将**听写语言**从“自动”更改为您的特定语言
    3.再次尝试听写
  </Step>
  
  <Step title="测试两种模式">
    尝试两种语音输入方式：
    - **按住说话**：按住`Fn`（或您的语音热键），说话，然后松开
    - **单击听写**：单击麦克风图标一次开始，再次单击完成
  </Step>
</Steps>

## 有效果吗？

尝试这个快速测试：
1. 按住语音热键（默认为`Fn`）
2. 说“你好，这是一个测试”
3. 松开按键
4. **成功：** 您应该看到提示框中出现“Hello, this is a test”

<Tip>
  **仍有问题？** 授予权限后尝试重新启动 Alter。有时 macOS 需要重新开始才能识别新权限。
</Tip>
