title: "自定义模型未显示"
description: "修复模型选择器中装载的 OpenAI、Mistral 或本地模型"
---

<Note>
  **重要提示：**您需要从事业（OpenAI、Mistral 等）获得 API 密钥，然后它们才会出现在 Alter 中。
</Note>

##发生了什么事？

您已添加API快捷键，但当您点击`/`查看模型时：
- **自定义**部分为空
- 您看不到最顶尖的型号
- 您只能看到改变云模型

## 常见的设置错误

<AccordionGroup>
  <Accordion icon="toggle-off" title="自定义提供程序已关闭">
    即使输入了API密钥，您也需要实现自定义提供程序：
    
1.打开Alter设置(`Cmd + ,`)
    2. 转到 **API 密钥** 选项卡
    3.找到**自定义创作**部分
    4.将**实现自定义提供程序**切换为开
    5. 您应该在“自定义端点”旁边看到一个 ✓
  </Accordion>
  
<Accordion icon="link" title="端点格式错误（尤其是对于 Azure）">
    **对于最重要的：**绑紧您的 API 挂钩 – 端点是自动的
    
**特别是对于 Azure**，您需要完整的 URL：
    ```
    https://your-instance.openai.azure.com/openai/deployments/your-model-name/chat/completions?api-version=2024-02-15-preview
    ```
    
确定您的网址包括：
    - 您的实例名称
    - 名称部署
    - api版本参数
  </Accordion>
  
  <Accordion icon="key" title="API 密钥无效或过期">
    快速检查：
    - 你复制了完整的密钥吗？ （它们通常以 `sk-` 开头）
    - 密钥是否已过期？
    - 您是否与提供商设置了账单？
    - 您是否不小心包含了额外的空格？
    
    尝试从提供商的仪表板生成新密钥。
  </Accordion>
</AccordionGroup>

## 快速设置指南

<Steps>
  <Step title="获取您的 API 密钥">
    - **OpenAI：** https://platform.openai.com/api-keys
    - **米斯特拉尔：** https://console.mistral.ai/api-keys/
    - **双子座：** https://aistudio.google.com/app/apikey
  </Step>
  
<Step title="将其输入 Alter">
    1. 设置(`Cmd + ,`) → **API密钥**
    2.从下拉列表中选择您的事业
    3.粘贴您的钥匙
    4.将**实现自定义提供程序**开放
  </Step>
  
<Step title="检查型号选择器">
    1、在更改提示框按`/`
    2. 查找 **自定义** 部分
    3.您应该会看到上市的最佳型号
  </Step>
</Steps>

## 验证它是否正常工作

设置后：
1.点击`/`打开模型选择器
2. 查找**自定义**类别
3. 选择您的定制型号之一
4.发送测试消息
5. **成功：**模型使用您自己的API按键进行响应！

<Tip>
  **预算提示：**使用您自己的API密钥意味着将直接向您报告。在他们的仪表板上监控您的使用情况统计意外！
</Tip>
