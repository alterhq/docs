## 使用限制

在合理使用下：
- 每日限制：`200`次请求/天
- 超限后：降速为`1 次请求 / 10 分钟`
- 预算稳定上限流量：可充值预算

<Note>
  API流量面向路由器个人和低工作负载。对于多用户产品或高吞吐的代理系统，请使用大型专用设施基础设施。
</Note>

## 常见问题

### 模型未上市
- 手动将模型设置为`Provider#Model-name`
- 检查API密钥是否有效
- 检查客户端是否支持模型列表端点

### 认证错误
- 确认`Authorization: Bearer YOUR_API_KEY`
- 确认终点是`https://alterhq.com/api` 或 `/api/v1`
- 确认账号后激活状态

###连接问题
- 使用OpenAI兼容客户端
- 检查`alterhq.com`的网络感受

### 高延迟
- 尝试更轻量模型（`OpenAI#gpt-5-nano`、`Alter#light`）
- 长输出使用流媒体

## 最佳实践

1. API key放在环境变量中（不要硬编码）
2. 监控请求量和错误率
3.让模型能力匹配任务复杂度
4. 实现带退避的重试
5.日常任务利用闲置模型
