このガイドで、Alter における最適な Web 検索構成を選べます。

<Info>
  まずは Alter ルーターを使ってください。`Fast`、`Best`、`Fair`、`Light` は Web 検索を標準搭載しています。
</Info>

## 使える選択肢

- **Local Tools:** `Web Search` と `Web Search Gemini`
- **Remote MCP Servers:** Tavily、LinkUp、Exa
- **Apps Gallery:** Firecrawl、LinkUp、Exa（workspace で利用可能な場合）
- **Alter ルーター:** `Fast`、`Best`、`Fair`、`Light` のネイティブ Web 検索

## プロバイダーの位置付け

- **Tavily:** AI agents 向けの Web アクセス層。search、extraction、research、crawling を単一 API で提供。
- **Firecrawl:** Web サイトを LLM-ready データへ変換。scraping と crawling を重視。
- **LinkUp:** AI apps 向け検索。standard/deep モードとソース付き回答を訴求。
- **Exa:** 品質重視の AI 検索 API。search から research まで複数 endpoint を提供。

## ハイブリッド戦略

1. まずはルーターのネイティブ検索から始めると、設定が最小で既定の結果も良好です。
2. Web 検索を持たないモデル（ローカルモデルを含む）には、Alter 同梱ツールで拡張します。外部アカウントは不要です。
3. どのモデルでも高い結果品質や詳細な制御が必要なら、MCP providers を追加します。
4. 複数ページの構造化抽出が必要なら scraper を追加します。

<Warning>
  無料枠や上限は変更されることがあります。利用前に最新情報を確認してください。
</Warning>

<Tip>
  Web 検索ツールを同時に複数有効化しないでください。モデルのツール選択が不安定になります。
</Tip>
