<Info>
  Das **Kontextfenster** ist die Menge an Text, die ein KI-Modell in einem einzigen Gespräch verarbeiten kann. Man kann es sich wie das Arbeitsgedächtnis des Modells vorstellen — es umfasst Ihre Nachrichten, Dateiinhalte, Tool-Definitionen und die Antworten der KI.
</Info>

## Warum das Kontextfenster wichtig ist

Jede Interaktion mit der KI verbraucht Kontextplatz. Wenn Ihr Kontextfenster voll ist:

<CardGroup cols={3}>
  <Card title="Leistung verlangsamt sich" icon="gauge">
    Größere Datenmengen brauchen länger zur Verarbeitung, was die Antwortzeiten erhöht.
  </Card>
  <Card title="Fair Use Budget erschöpft sich" icon="scale-balanced">
    Mehr Tokens pro Anfrage verbrauchen Ihr Fair Use-Policy-Budget schneller.
  </Card>
  <Card title="Kontext wird abgeschnitten" icon="scissors">
    Ältere Nachrichten können verworfen werden, wodurch die KI frühere Teile des Gesprächs "vergessen" kann.
  </Card>
</CardGroup>

Zu verstehen, wie man das Kontextfenster verwaltet, hilft Ihnen, eine bessere Leistung und mehr Wert aus Alter zu ziehen.

---

## Was Kontextplatz verbraucht

### 1. Gesprächsverlauf

Jede Nachricht, die Sie senden, und jede Antwort, die Sie erhalten, bleibt im Kontext. Lange Gespräche verbrauchen natürlicherweise mehr Platz.

### 2. Dateiinhalte

Wenn Sie Dateien oder Ordner anhängen, werden deren gesamte Inhalte in den Kontext eingefügt:

- **Textdateien** — Vollständiger Inhalt ist enthalten
- **Code-Dateien** — Der gesamte Dateiinhalt wird gesendet
- **Dokumente** — Analysierter Text ist enthalten
- **Bilder** — Beschreibungen oder OCR-Text werden hinzugefügt

### 3. Tool-Definitionen

Jedes aktivierte Tool verbraucht Kontextplatz. Eine Tool-Definition beinhaltet:

- Tool-Name und Beschreibung
- Erforderliche und optionale Parameter
- Parameterbeschreibungen und -typen

<Warning>
  **Tools sind die stillen Kontextverbraucher.** Die Aktivierung von 10+ Tools kann leicht 30-50% Ihres Kontextfensters verbrauchen, bevor Sie überhaupt mit dem Tippen beginnen.
</Warning>

### 4. Systemanweisungen

Hintergrundaufforderungen, Grundlagendokumente und Systemanweisungen verbrauchen ebenfalls Platz.

---

## Wie Tools Kontext verbrauchen

Hier ist ein reales Beispiel, wie Tools Ihren Kontext beeinflussen:

| Aktivierte Tools | Ungefähr verbrauchter Kontext | Verbleibend für Ihre Inhalte |
|----------------|-------------------------------|------------------------------|
| 0 Tools | ~5% | ~95% |
| 5 Tools | ~15% | ~85% |
| 10 Tools | ~30% | ~70% |
| 20 Tools | ~50% | ~50% |
| 30+ Tools | ~70%+ | ~30% |

<Note>
  Dies sind ungefähre Werte. Die tatsächliche Nutzung variiert je nach Komplexität der Tools und der Länge der Beschreibungen.
</Note>

## Strategien zur Verwaltung des Kontexts

### 1. Verwenden Sie Flow zur Tool-Orchestrierung (Empfohlen)

Die effektivste Möglichkeit, den kontextbezogenen Verbrauch durch Tools zu reduzieren, besteht darin, **Flow** zu verwenden — Alters integrierten Tool-Orchestrator.

<Card title="Flow lädt als einzelnes Tool" icon="wand-magic-sparkles" href="/de/guides/use-flow-orchestration">
  Anstatt alle aktivierten Tools in jede Anfrage zu laden, fungiert Flow als ein Tool, das intelligent die richtigen Tools für jede Aufgabe auswählt und orchestriert.
  
  **Vorteile:**
  - Reduziert den Kontextverbrauch um 60-80%
  - Verbessert die Antwortzeiten
  - Handhabt automatisch mehrstufige Aufgaben
  - Spart das Fair Use-Budget
</Card>

**Beste Praxis:** Konfigurieren Sie Ihre standardmäßige "Ask Anything"-Aktion so, dass nur Flow anstelle aller aktivierten Tools verwendet wird.

### 2. Deaktivieren Sie ungenutzte Tools

Aktivieren Sie nur die Tools, die Sie aktiv verwenden:

<Steps>
  <Step title="Überprüfen Sie Ihre Tools">
    Gehen Sie zu **Einstellungen > Tool-Manager** und sehen Sie, was aktiviert ist
  </Step>
  <Step title="Deaktivieren Sie ungenutzte Tools">
    Schalten Sie Tools aus, die Sie in der vergangenen Woche nicht verwendet haben
  </Step>
  <Step title="Bei Bedarf wieder aktivieren">
    Sie können Tools immer wieder aktivieren — sie merken sich Ihre Authentifizierung
  </Step>
</Steps>

### 3. Passen Sie Tool-Sets für Aktionen an

Jede Aktion kann mit ihren eigenen spezifischen Tools konfiguriert werden. Dies ist eine leistungsstarke Möglichkeit, den Kontextverbrauch zu kontrollieren:

#### Ask Anything Aktion

Die **Ask Anything**-Aktion ist die standardmäßig verwendete Aktion, wenn Sie in das Eingabefeld tippen, ohne eine bestimmte Aktion auszuwählen. Sie läuft überall in Alter — minimieren Sie Ihre Tool-Auswahl hier für maximale Wirkung.

<Steps>
  <Step title="Öffnen Sie den Aktions-Editor">
    Gehen Sie zu **Einstellungen > Aktionen**
  </Step>
  <Step title="Finden Sie Ask Anything">
    Wählen Sie die Aktion "Ask Anything" aus
  </Step>
  <Step title="Klicken Sie auf Erweitert">
    Erweitern Sie den Abschnitt Erweitert
  </Step>
  <Step title="Begrenzen Sie Tools">
    Wählen Sie nur die Tools aus, die Sie unbedingt benötigen (oder nur Flow)
  </Step>
</Steps>

#### Benutzerdefinierte Aktionen

Wenn Sie benutzerdefinierte Aktionen erstellen, denken Sie sorgfältig darüber nach, welche Tools tatsächlich benötigt werden:

<CardGroup cols={2}>
  <Card title="Lernen von Standardwerten" icon="book-open">
    Die meisten integrierten Alter-Aktionen haben standardmäßig keine Tools aktiviert. Zum Beispiel benötigt "Korrekte Grammatik" keine Tools — es ist eine reine Texttransformation.
  </Card>
  <Card title="Minimalistisch sein" icon="filter">
    Aktivieren Sie nur die Tools, die Ihr spezifischer Workflow erfordert. Wenn Ihre Aktion nur Text formatiert, benötigen Sie wahrscheinlich keine Tools.
  </Card>
</CardGroup>

#### Flow vs. spezifische Tools

<Info>
  **Verwenden Sie Flow**, wenn Sie Flexibilität wünschen und nicht wissen, welche spezifischen Tools Sie benötigen. Es übernimmt automatisch die Tool-Auswahl.
</Info>

<Info>
  **Verwenden Sie spezifische Tools**, wenn Sie genau wissen, welche Tools Ihr Workflow benötigt. Dies sorgt für eine bessere Leistung und weniger Hin und Her mit der KI.
</Info>

**Beispiel:** Wenn Sie eine Aktion erstellen, die nur Kalenderereignisse erstellt, aktivieren Sie nur das Kalender-Tool anstelle von Flow. Dies eliminiert den Schritt der Tool-Auswahl und führt schneller aus.

### 4. Verwenden Sie Workspaces für große Dateien

Anstatt große Dateien an jedes Gespräch anzuhängen:

- Erstellen Sie einen **Workspace** für Projekte mit großen Dateien
- Verweisen Sie auf den Workspace, wenn nötig
- Löschen Sie den Workspacekontext beim Wechseln von Aufgaben

<Card title="Erfahren Sie mehr über Workspaces" icon="folder-open" href="/de/guides/core-features#workspaces">
  Workspaces helfen dabei, Dateien zu organisieren und den Kontext effizienter zu verwalten
</Card>

### 5. Kontext regelmäßig löschen

Verwenden Sie **⌘ K** (Befehl + K), um alle Kontexte zu löschen, wenn:

- Sie eine neue, nicht verwandte Aufgabe starten
- Der vorherige Kontext nicht mehr relevant ist
- Die Antworten langsam werden

### 6. Lange Gespräche zusammenfassen

Für sehr lange Gespräche:

- Exportieren Sie das Gespräch in eine Datei
- Starten Sie ein neues Gespräch mit einer Zusammenfassung
- Verweisen Sie auf die exportierte Datei, wenn nötig

### 7. Wählen Sie Dateien selektiv aus

Anstatt ganze Ordner anzuhängen:

- Hängen Sie nur die spezifischen Dateien an, die Sie benötigen
- Entfernen Sie Dateien aus dem Kontext, wenn Sie mit ihnen fertig sind

## Überwachung des Kontextverbrauchs

Obwohl Alter keinen Echtzeit-Kontextmesser anzeigt, können Sie die Nutzung indirekt überwachen:

<CardGroup cols={2}>
  <Card title="Antwortzeiten beobachten" icon="clock">
    Langsame Antworten deuten oft auf einen hohen Kontextverbrauch hin.
  </Card>
  <Card title="Überprüfen Sie die Toolanzahl" icon="toolbox">
    Mehr aktivierte Tools = mehr verbrauchter Kontext. Überprüfen Sie den Tool-Manager regelmäßig.
  </Card>
</CardGroup>

## Schnellgewinne-Checkliste

<Steps>
  <Step title="Flow aktivieren" icon="check">
    Konfigurieren Sie "Ask Anything", um nur Flow anstelle aller Tools zu verwenden
  </Step>
  <Step title="Aktionen anpassen" icon="check">
    Setzen Sie spezifische Tools für benutzerdefinierte Aktionen — deaktivieren Sie Tools, die Sie nicht benötigen
  </Step>
  <Step title="Tools überprüfen" icon="check">
    Deaktivieren Sie Tools, die Sie kürzlich nicht verwendet haben
  </Step>
  <Step title="Workspaces verwenden" icon="check">
    Organisieren Sie große Projekte in Workspacesn, anstatt Dateien ad-hoc anzuhängen
  </Step>
  <Step title="Kontext löschen" icon="check">
    Drücken Sie **⌘ K**, wenn Sie neue Aufgaben beginnen
  </Step>
</Steps>
