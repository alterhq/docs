title: "Utilice Flow para la orquestación Tool"
description: "Deje que Flow seleccione y organice automáticamente sus Tools para reducir el uso del contexto y mejorar el rendimiento."
---

<Info>
**Flow** es el orquestador de Tools integrado de Alter. En lugar de cargar todas sus Tools habilitadas en cada solicitud, Flow actúa como una única Tool que dirige inteligentemente sus solicitudes a las Tools adecuadas.
</Info>

<Warning>
**Habilite Tools primero**: Flow solo puede usar Tools que ya haya habilitado en Tool Manager. No habilita automáticamente las Tools deshabilitadas. Asegúrese de que las Tools que necesita estén activadas antes de usar Flow.
</Warning>

## ¿Por qué utilizar Flow?

<CardGroup cols={3}>
  <Card title="Ahorra espacio de contexto" icon="compress">
Flow se carga como una única Tool en lugar de todas las Tools habilitadas, lo que libera la ventana contextual para el contenido real.
  </Card>
  <Card title="Rendimiento más rápido" icon="bolt">
Las cargas útiles más pequeñas se procesan más rápido y reducen la latencia en cada solicitud
  </Card>
  <Card title="Ahorre el presupuesto de uso legítimo" icon="scale-balanced">
Menos tokens por solicitud significa menos uso en comparación con el presupuesto de su política de uso justo
  </Card>
</CardGroup>

Cada Tool que habilita consume espacio en su ventana contextual. Cuando tienes más de 10 Tools habilitadas, esto afecta significativamente el rendimiento. Flow resuelve este problema actuando como una única Tool que organiza todas las demás.

---

## Habilitar Flow

<Steps>
  <Step title="Abrir configuración">
Presione **⌘,** (Comando + Coma) o haga clic en **Configuración** en el menú
  </Step>
  <Step title="Ir a Tool Administrador">
Vaya a la pestaña **Tool Administrador**
  </Step>
  <Step title="Habilitar Flow">
En **Local Tools > Alter**, active **Flow**
  </Step>
</Steps>

<Note>
Recuerde: Flow solo puede orquestar Tools que ya están habilitadas. Habilite las Tools específicas que necesita (Calendario, Gmail, Slack, etc.) además del propio Flow.
</Note>

---

## Cómo funciona Flow

Cuando envía una solicitud con Flow habilitado, sigue este proceso:

<Steps>
  <Step title="Analizar">
Flow analiza su solicitud para comprender lo que está tratando de lograr
  </Step>
  <Step title="Seleccionar">
De sus Tools habilitadas, Flow selecciona las Tools apropiadas para la tarea
  </Step>
  <Step title="Ejecutar">
Flow ejecuta las llamadas a la Tool en secuencia, pasando datos entre pasos según sea necesario
  </Step>
  <Step title="Resumir">
Los resultados se condensan en una respuesta concisa en la que se conservan los puntos de datos clave.
  </Step>
</Steps>

---

## Ejemplos

<Tabs>
  <Tab title="Único Tool">
**Solicitud:** "¿Qué reuniones tengo hoy?"
    
Flow automáticamente:
    1. Reconoce que necesita información del calendario
    2. Selecciona la Tool Calendario.
    3. Recupera tus reuniones
    
No es necesario seleccionar manualmente la Tool Calendario: Flow se encarga de ello.
  </Tab>
  
  <Tab title="Cadena múltiple-Tool">
**Solicitud:** "Cancelar mi reunión de las 3:00 p. m. y enviar un correo electrónico a los asistentes explicando que estoy enfermo"
    
Flow automáticamente:
    1. Utiliza la Tool **Calendario** para encontrar su reunión de las 3 p.m.
    2. Identifica a los asistentes de la reunión.
    3. Utiliza la Tool **Gmail** para redactar un correo electrónico.
    4. Envía la cancelación y la explicación.
    
Todo en una solicitud en lenguaje natural.
  </Tab>
  
  <Tab title="Con Transformación">
**Solicitud:** "Elimine la documentación Alter y extraiga las características clave"
    
Flow automáticamente:
    1. Utiliza una Tool web para buscar el contenido de la documentación.
    2. Transforma el HTML sin formato usando `$1:Extract the key features and format as bullet points`
    3. Devuelve una lista formateada de características.
    
El paso de transformación extrae solo información relevante.
  </Tab>
</Tabs>

## Cuándo utilizar Flow frente a Tools específico

<CardGroup cols={2}>
  <Card title="Utilice Flow" icon="wand-magic-sparkles">
    <ul>
      <li>You're not sure which tool to use</li>
      <li>You want simple, natural language requests</li>
      <li>You have many tools enabled</li>
      <li>You're doing complex multi-step tasks</li>
    </ul>
  </Card>
  <Card title="Utilice acciones Tool específicas" icon="bullseye">
    <ul>
      <li>You know exactly which tool you need</li>
      <li>You want the fastest possible execution</li>
      <li>You're building repeatable workflows</li>
      <li>You need predictable, consistent behavior</li>
    </ul>
  </Card>

<Note>
**Regla del 90%:** Para el 90% de las tareas, Flow es la mejor opción. Cree acciones de Tools específicas solo cuando necesite flujos de trabajo optimizados y repetibles.
</Note>

## Mejores prácticas: configurar "Preguntar cualquier cosa"

La acción **Preguntar cualquier cosa** es la acción predeterminada de Alter: se usa cuando escribe en el cuadro de diálogo sin seleccionar una acción específica.

<Steps>
  <Step title="Abrir editor de acciones">
Vaya a **Configuración > Acciones**
  </Step>
  <Step title="Buscar Preguntar cualquier cosa">
Busque la acción **"Preguntar cualquier cosa"**
  </Step>
  <Step title="Haga clic en Avanzado">
Expanda la sección **Avanzado**
  </Step>
  <Step title="Seleccione sólo Flow">
En la sección **Tools**, seleccione solo **Flow** (anule la selección de todas las demás Tools)
  </Step>
  <Step title="Ahorrar">
Guarda tus cambios
  </Step>
</Steps>

Ahora sus interacciones predeterminadas utilizarán la orquestación de Flow en lugar de cargar todas las Tools habilitadas en cada solicitud.

---

<AccordionGroup>
  <Accordion icon="link" title="Avanzado: Tool Encadenamiento con instrucciones">
Flow encadena automáticamente Tools cuando una tarea requiere varios pasos. Se puede hacer referencia al resultado de una Tool en llamadas posteriores utilizando la sintaxis `$N`.

    ### Sintaxis de referencia

| Sintaxis | Descripción |
    |--------|-------------|
| `$1` | Salida bruta del paso 1 |
| `$2` | Salida bruta del paso 2 |
| `$N:instruction` | Transforme la salida del paso N usando una instrucción |

    ### Ejemplo con transformación

Al raspar un sitio web y formatear los resultados:

    1. Flow utiliza una Tool web para buscar contenido
    2. Aplica transformación: `$1:Extract the key frameworks and format as bullet points`
    3. El LLM procesa el contenido sin procesar antes de pasar a la siguiente Tool.

Esto permite a Flow:
    - Extraiga solo información relevante de resultados grandes
    - Formatee los datos adecuadamente para la siguiente Tool
    - Reducir el ruido en los pasos siguientes.
  </Accordion>

  <Accordion icon="memory" title="Avanzado: gestión inteligente del contexto">
Flow incluye un paso de resumen integrado que optimiza la forma en que se devuelven los resultados a su conversación.

    ### ¿Qué hace el resumen?

Después de ejecutar las Tools, Flow:

    1. **Condensa la salida**: comprime las salidas detalladas de la Tool
    2. **Conserva datos importantes**: conserva ID, URL, referencias y puntos de datos
    3. **Resalta ideas clave**: destaca enlaces, ideas y hallazgos para su seguimiento.
    4. **Proporciona citas de fuentes**: utiliza la notación [Paso N] para la trazabilidad

    ### Por qué esto importa

Sin resumen, los flujos de trabajo complejos inflarían su ventana de contexto. En el turno 5 o 6, alcanzarías los límites de fichas.

El resumen de Flow garantiza:
    - **Contexto eficiente**: solo se conserva la información esencial
    - **Respuestas procesables**: se conservan los puntos de datos importantes
    - **Mejor rendimiento**: menos contexto = respuestas más rápidas
    - **Trazabilidad** — [Paso N] las citas verifican las fuentes de información

    ### Salida de ejemplo

    ```
    Task completed successfully.

    Key artifacts for next actions:
    - Meeting ID: 12345 [Step 1]
    - Attendees: alice@example.com, bob@example.com [Step 1]
    - Draft email prepared [Step 2]

    Important findings:
    - Meeting conflicts with another event at 3pm [Step 1]
    - Alice has an out-of-office message active [Step 2]
    ```
  </Accordion>
</AccordionGroup>

## Solución de problemas

<AccordionGroup>
  <Accordion icon="xmark" title="Flow Seleccionó el Tool incorrecto">
Reformule su solicitud con detalles más específicos:
    
    - **En lugar de:** "Revisar mis mensajes"
    - **Prueba:** "Revisa mis mensajes de Slack no leídos en el canal general"
    
Cuanto más específico sea, mejor podrá Flow hacer coincidir su intención con la Tool adecuada.
  </Accordion>
  
  <Accordion icon="question" title="Flow No puedo encontrar un Tool">
Asegúrese de que la Tool sea:
    - **Habilitado** en Tool Manager (Flow no puede usar Tools deshabilitadas)
    - **Configurado correctamente** (p. ej., autenticado con Gmail)
    - **No restringido** por tu configuración de acción
  </Accordion>
  
  <Accordion icon="gauge-high" title="El rendimiento sigue siendo lento">
Incluso con Flow, las solicitudes muy grandes pueden resultar lentas. Intentar:
    - Dividir tareas muy complejas en 2 o 3 pasos
    - Usar acciones de Tools específicas para tareas que se repiten con frecuencia
    - Habilitar solo Tools esenciales en Tool Manager
  </Accordion>
</AccordionGroup>
